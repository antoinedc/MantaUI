# bui mobile — Capacitor wrapper design

Date: 2026-05-16
Status: Approved

## Goal

Wrap the existing `src/server/public/` mobile web client in a Capacitor native
shell so it can be distributed via TestFlight (iOS) and direct APK (Android).
The app is a pure remote viewer: it connects over WebSocket to the user's own
bui server running on their Linux box. No code runs on the device.

## Constraints (hard "do not")

- Do **not** modify `src/server/index.mjs`, `src/server/public/*`, or any
  Electron code (`src/main/`, `src/renderer/`).
- No build step (no webpack/vite) for the web assets. Plain JS only.
- No React/framework. No PWA manifest or service worker.
- `mobile/` is self-contained and must not affect existing root npm scripts.

## Toolchain reality (this Mac, 2026-05-16)

- iOS: **only** Xcode Command Line Tools installed; no `Xcode.app`, no
  CocoaPods. iOS platform can be *scaffolded* but **cannot be built/synced**
  here (`cap sync ios` runs `pod install`). A real `.ipa` needs interactive
  Xcode archiving + Apple Developer account regardless.
- Android: full toolchain present — cmdline-tools, platform-tools, Android
  Studio, JDK 17. A debug `.apk` **can** be produced headlessly via Gradle.

Decision: **Android is the concrete build deliverable.** iOS is scaffolded and
configured (Info.plist) and the build steps are documented for the user to run
once `Xcode.app` + CocoaPods are installed.

## Repo structure after this work

```
better-ui/
  mobile/
    .gitignore           # ignores node_modules/ and www/vendor/
    package.json
    capacitor.config.json
    sync-web.sh           # copies vendor files from root node_modules
    www/                  # Capacitor web dir
      index.html          # forked + committed (modified copy)
      app.js              # forked + committed (modified copy)
      vendor/             # GENERATED by sync-web.sh, gitignored
    ios/                  # scaffolded Xcode project (committed)
    android/              # Android Studio project (committed)
  src/server/             # UNCHANGED
  ...
```

## www/ sourcing strategy

`mobile/www/index.html` and `mobile/www/app.js` are **forked, hand-modified,
committed** copies of `src/server/public/*`. They diverge from the originals
(server-URL config screen, iOS keyboard fixes), so a copy-and-patch script
would be fragile. `sync-web.sh` only manages the regenerable vendor files.

## Web client modifications

### Settings / server-URL screen

- New `#settings` view in `index.html` (sibling of `#list` and `#term`):
  a heading, a `type="url"` input (`#server-url`, placeholder
  `http://myserver:8787`), a "Connect" button, and an inline error slot.
- localStorage key: `bui_server`.
- Boot logic in `app.js`:
  - If `bui_server` unset/blank → show `#settings`, hide list.
  - Else → normal boot (`loadSessions()`).
- "Connect" trims the input, normalizes (strip trailing `/`), validates it
  parses as an `http(s)://` URL, saves to localStorage, hides settings, runs
  `loadSessions()`. Invalid input shows the inline error, no save.
- Gear button (`⚙`) added to the list header next to refresh; opens `#settings`
  pre-filled with the current value so it can be changed.

### serverBase() helper

- `serverBase()` returns the saved base with no trailing slash.
- All `fetch()` calls become `serverBase() + "/api/..."`.
- WebSocket URL: derive proto from the saved base — `https:` → `wss:`,
  otherwise `ws:` — and use the saved host/port, not `location.host`.
  i.e. `serverBase().replace(/^http/, "ws") + "/pty?..."`.

### iOS keyboard / viewport fixes (apply to www/ copy only)

In `openTerm()`, after `term.open()` + `fit.fit()`:

- `touchend` listener on `#term-host` that synchronously calls
  `term._core.textarea.focus({ preventScroll: true })` so the iOS virtual
  keyboard appears on tap inside WKWebView.
- After xterm init, set on `term._core.textarea`:
  `autocorrect="off"`, `autocapitalize="off"`, `spellcheck="false"`.
- Extend the existing `visualViewport` resize path so it also resizes
  `#term-host` to `visualViewport.height − .term-bar height − .term-keys
  height` before `fit.fit()` and the resize message. This folds into the
  existing `onResize` rather than adding a second listener (the original
  already wires `visualViewport`; we add the height computation).

These changes are inert on Android/desktop (no `visualViewport` keyboard
overlap there; `touchend` focus is harmless).

## sync-web.sh

Bash script, run from anywhere, resolves repo root relative to its own path.
Copies into `mobile/www/vendor/`:

- `node_modules/@xterm/xterm/lib/xterm.js` → `vendor/xterm.js`
- `node_modules/@xterm/xterm/css/xterm.css` → `vendor/xterm.css`
- `node_modules/@xterm/addon-fit/lib/addon-fit.js` → `vendor/addon-fit.js`

Fails loudly if a source file is missing (root deps not installed). The
existing `index.html` vendor paths are already `/vendor/*`, so no path edits.

## Capacitor configuration

`capacitor.config.json`:

```json
{
  "appId": "com.antoinedc.bui",
  "appName": "bui",
  "webDir": "www",
  "server": { "allowNavigation": ["*"] }
}
```

Init flow:

```
cd mobile
npm init -y
npm install @capacitor/core @capacitor/ios @capacitor/android
npm install -D @capacitor/cli @capacitor/assets
npx cap init "bui" "com.antoinedc.bui" --web-dir www
npx cap add android
npx cap add ios            # scaffold only; sync deferred (no CocoaPods here)
```

`mobile/package.json` scripts:

```json
{
  "scripts": {
    "sync": "bash sync-web.sh && npx cap sync android",
    "sync:ios": "bash sync-web.sh && npx cap sync ios",
    "android": "npx cap open android",
    "ios": "npx cap open ios",
    "apk": "bash sync-web.sh && npx cap sync android && cd android && ./gradlew assembleDebug"
  }
}
```

Note `npm run sync` syncs **android only** so it succeeds on this Mac. A
separate `sync:ios` is documented for once CocoaPods is installed.

### iOS config (scaffolded, not built here)

- `mobile/ios/App/App/Info.plist`: add `NSAppTransportSecurity` →
  `NSAllowsArbitraryLoads = true` (lets the app reach user HTTP servers).
- App Store / TestFlight description (documented, not code):
  "bui connects to your own remote Linux server to display tmux terminal
  sessions. All code execution is on your server; the app is a remote viewer."

### Android config

- `mobile/android/app/src/main/AndroidManifest.xml`:
  - `android:usesCleartextTraffic="true"` on `<application>`.
  - `<uses-permission android:name="android.permission.INTERNET" />`
    (Capacitor adds this by default; verify and add if absent).

## App icon

Generate a 1024×1024 placeholder PNG (dark background `#0e0f12`, accent
`#66c0f4` terminal-style `bui` / prompt glyph). Place at
`mobile/assets/icon.png` and run `npx capacitor-assets generate` (via
`@capacitor/assets`) to populate Android mipmaps (and iOS AppIcon set when
that platform is built). Placeholder is swappable later.

## Verification

Realistic given the toolchain:

1. `cd mobile && npm run sync` → completes with no error; `www/vendor/` has the
   3 files; `android/` synced.
2. `npm run apk` → Gradle `assembleDebug` succeeds; report the produced
   `mobile/android/app/build/outputs/apk/debug/app-debug.apk` path + size.
3. iOS: `ios/` project exists, `Info.plist` contains `NSAllowsArbitraryLoads`.
   Document the deferred steps (install Xcode.app + `sudo gem install
   cocoapods`, then `npm run sync:ios`, open in Xcode, Archive → TestFlight).
4. Manual (user, post-build): install APK on a device/emulator → settings
   screen appears → enter a valid bui server → session list loads → tap a
   session → xterm renders → on-screen keys + soft keyboard work.

## Git

- Add `mobile/.gitignore`: `node_modules/`, `www/vendor/`.
- Commit `mobile/` including `ios/` and `android/` project dirs and the
  generated icon assets. Do not commit `mobile/node_modules` or
  `mobile/www/vendor`.
- Atomic commits per logical step.

## Out of scope

- Real signing / Play Store release config (debug APK only).
- iOS build/archive on this machine.
- Any change to the bui server or Electron app.
- Designed branding/logo (placeholder icon only).
