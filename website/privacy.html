Privacy

Plain-language summary of what Manta UI stores where, who can read it, and how to delete it.

Manta UI is a thin client. The interesting data lives on your Linux box; the relay only sees the parts that have to cross the public internet to reach your phone.

What runs where

Manta UI is two surfaces over the same protocol:

  • The desktop app (Electron) and the phone client (PWA today, native apps in flight) are thin clients — chat, terminal, and file browser over HTTPS.
  • The server runs on your own Linux box. It owns your tmux sessions, your opencode chat sessions, your file uploads, your schedules, your secrets, your webhooks, and your notifications. Everything lives under ~/.manta/ on that box.
  • The relay (relay.mantaui.com) is the operated backend that routes phone traffic to your box. Your box dials out to the relay over a WebSocket; nothing on the box needs an inbound port.

What's on your box

  • Box identity — a random box_id (32 hex chars) and a box_token (HMAC key) under ~/.manta/auth.json (0600). The token never leaves the box.
  • Pairing codes — short-lived (5 min), loopback-only, one-time. Mints when you run npm run pair or hit GET /auth/pair from the box itself.
  • Device tokens — issued to each paired desktop / phone. Stored locally in each app's config.json; the box keeps no per-device record beyond "is this token currently valid."
  • Your work — every tmux session, every opencode chat, every uploaded file (~/.manta-uploads/, hourly auto-clean), every scheduled prompt, every webhook, every secret, every published page.

Delete any of this by deleting the file. The installer's only hard rule is: never regenerate ~/.manta/auth.json — that would unpair every device.

What's on the relay

The relay holds the minimum it needs to route traffic and (when subscription lands) gate paid endpoints. It does not proxy your terminal bytes through anything except the encrypted box↔relay WebSocket.

  • Box row — box_id, first-seen, last-seen, online/offline status.
  • Account binding — which account owns which box (one account ↔ many boxes; a box belongs to exactly one account).
  • Device tokens — phone tokens stored as sha256(token); the plaintext is handed to the phone once at /pair time and never persisted server-side.
  • Box handshake credential — sha256(box_token), trust-on-first-use (a second box claiming the same box_id is rejected).
  • Native push tokens — APNs or FCM device tokens, so the relay can deliver a push when the box is on a different network than the phone. (Stage 5; not yet wired for production.)
  • IAP receipts — Apple in-app-purchase receipts bound to a box_id. (Stage 5; in place, awaiting production deployment.)
  • Byte-usage counters — per-box ingress and egress totals, the COGS signal the subscription meter reads.

Retention is until you unbind the box (the relay drops the binding row + cascades the dependents) or delete the box. The relay has no concept of a "marketing profile" — it stores nothing about you beyond what's needed to route your traffic.

What crosses the wire

  • Phone → box: chat prompts, terminal keystrokes, file uploads. Authenticated with your device token; the box checks HMAC + expiry on every request.
  • Box → relay: a single outbound WebSocket to wss://relay.mantaui.com/box/<box_id>, authenticated with your box_token. The relay cannot read the contents (it just forwards frames).
  • Relay → phone: chat responses, terminal output, file download responses, push notifications. Web Push today (VAPID, PWA); native APNs/FCM in flight for the iOS / Android apps.

Push notifications

When you have the phone client open in the background (or installed as a native app later), Manta may send a push when an agent needs you — a permission prompt, a question, an error, or a long-running task finishing. The push body carries the session name and a short snippet of the question. We do not include the full transcript in the push body.

Cookies and analytics on this site

This website (the one you're reading right now) does not set cookies, does not run analytics, does not embed trackers, and does not fingerprint your browser. The only outbound requests are to Google Fonts for typography.

Third parties we use

  • Google Fonts — typography CSS, loaded from fonts.googleapis.com. Google sees your IP. If you want to avoid that, the page degrades gracefully to system fonts.
  • Apple Push Notification service (APNs) / Firebase Cloud Messaging (FCM) — for native push on the iOS / Android apps. Apple and Google see the push envelope and your IP.
  • Stripe / Apple IAP — when the subscription ships, payment is handled by the app store; we never see your card number.

Open source

Manta UI is open source. The server, relay, mobile web client, and desktop app are all on GitHub. The only closed-source piece is the published iOS / Android app (which exists only because the stores require it).

Your choices

  • Run the server yourself. Every user runs their own server on their own hardware. You can point a phone at your own box directly, skip the relay entirely, and not share any data with us.
  • Use the relay. If you go through relay.mantaui.com, you accept the relay's storage of the fields above. Unpair any time from your box: delete the device from the desktop app, or wipe ~/.manta/.
  • Delete your data. Uninstalling the desktop or phone app removes its local config. Wiping ~/.manta/ on the box removes identity + work. Asking us to forget a box drops the relay row + cascades.
